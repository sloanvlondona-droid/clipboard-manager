# 📋 剪贴板历史管理器

一个 Windows 上的永久剪贴板历史记录工具，常驻系统托盘，自动记录所有复制内容。

---

## ✨ 功能

- **自动记录** — 实时监控剪贴板，自动保存文本、图片、文件路径
- **搜索过滤** — 输入关键词快速查找历史记录
- **置顶收藏** — 重要内容置顶，清空时不会被删除
- **右键菜单** — 列表项支持右键操作：复制、查看详情、置顶、删除
- **详情弹窗** — 查看完整文本内容或图片大图
- **系统托盘** — 后台常驻运行，左键单击/双击切换窗口
- **开机自启** — 托盘菜单一键开关，通过注册表实现
- **现代主题** — 白色简洁 UI，自定义滚动条、卡片式列表
- **单文件打包** — PyInstaller 打包为单个 exe（约 44MB）

## 🚀 快速开始

### 环境要求

- Windows 10 / 11
- Python 3.10+

### 安装依赖

```bash
pip install -r requirements.txt
```

### 运行

```bash
python main.py
```

程序启动后会在系统托盘显示一个剪贴板图标，点击即可打开历史窗口。

### 打包为 exe

```bash
pyinstaller --onefile --windowed --name "剪贴板历史管理器" main.py
```

或使用已有的 spec 文件：

```bash
pyinstaller 剪贴板历史管理器.spec
```

## 📁 项目结构

```
├── main.py                 # 程序入口，组装各模块并启动
├── main_window.py          # 主窗口 UI（列表、搜索、操作、详情弹窗）
├── clipboard_monitor.py    # 剪贴板监控（文本/图片/文件检测）
├── database.py             # SQLite 数据库（增删改查、搜索、图片存储）
├── tray_manager.py         # 系统托盘（图标、菜单、自启开关）
├── styles.py               # 全局样式（配色、卡片、按钮）
├── autostart.py            # 开机自启（注册表读写）
├── requirements.txt        # Python 依赖
├── 剪贴板历史管理器.spec    # PyInstaller 打包配置
├── clipboard_history.db    # SQLite 数据库文件（自动生成）
└── clipboard_images/       # 图片存储目录（自动生成）
```

## 🛠️ 技术栈

| 技术 | 用途 |
|------|------|
| **PyQt6** | GUI 框架，窗口、托盘、剪贴板访问 |
| **SQLite** | 本地持久化存储，WAL 模式 |
| **Pillow** | 图片读取与保存（ImageGrab） |
| **PyInstaller** | 打包为独立 exe |
| **Windows Registry** | 开机自启管理 |

## ⌨️ 操作说明

| 操作 | 方式 |
|------|------|
| 显示/隐藏窗口 | 左键单击托盘图标 |
| 聚焦窗口 | 左键双击托盘图标 |
| 搜索历史 | 顶部搜索框输入关键词 |
| 复制内容 | 左键单击列表项（自动复制到剪贴板） |
| 查看更多操作 | 右键点击列表项 |
| 查看详情 | 右键 → 查看详情 |
| 切换置顶 | 点击「📌 置顶」按钮 |
| 删除记录 | 点击「🗑 删除」按钮 |
| 清空全部 | 底部「🗑 清空全部」（保留置顶项） |
| 开关自启 | 右键托盘 → 🚀 开机自启 |
| 退出程序 | 右键托盘 → ❌ 退出 |

---

## 📝 数据库说明

- 使用 SQLite，文件位于程序目录 `clipboard_history.db`
- 文本直接存储在数据库中
- 图片保存到 `clipboard_images/` 目录，数据库记录文件路径
- 重复内容不会重复存储，只更新使用时间
- 置顶记录在「清空全部」时会被保留
